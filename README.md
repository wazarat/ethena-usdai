# Dune Dashboard Builder — Ethena vs USD.AI

A self-contained TypeScript/Node agent that builds the **Ethena vs USD.AI** Dune
dashboard on a **Free Dune plan**. It does the parts the Free API allows
(discovery, verification) and hands you an exact, click-by-click build sheet for
the one manual UI step the Free plan requires.

## What's automatable on Free (and what isn't)

| Task | Free plan? | Who |
|---|---|---|
| Read results of existing public queries | ✅ | **Agent** |
| Read your `dune.wazarat.canhav_agent_verdicts` table (via a saved query) | ✅ | **Agent** |
| Create a NEW saved query via API | ❌ paid | — |
| Create / assemble a dashboard via API | ❌ paid | — |
| GitHub dashboard sync ("Turn dashboard into a repo") | ❌ Plus+ | — |

So this tool removes the guesswork: it **verifies** your query IDs and columns,
then emits `BUILD_SHEET.md` with everything pre-filled. The final dashboard
assembly is a short manual UI step.

> The agent never creates queries/dashboards via API, never logs your API key,
> and uses only the latest-results endpoint (no execution credits).

## Setup

1. **Rotate** your Dune API key (the previous one was exposed) at
   <https://dune.com/settings/api>.
2. Copy the env template and fill it in:
   ```bash
   cp .env.example .env
   # edit .env -> set DUNE_API_KEY (and optionally the query IDs)
   ```
3. Install dependencies:
   ```bash
   npm install
   ```

## Usage

Guided end-to-end flow (prompts for any query IDs not already in `.env`):

```bash
npx tsx build.ts
# or: npm run build
```

It will:
1. Collect the two supply query IDs (Ethena sUSDe, USD.AI sUSDai).
2. Verify each runs and detect its date + value columns.
3. Verify your verdicts source (optional query ID over `dune.wazarat.canhav_agent_verdicts`).
4. Write `BUILD_SHEET.md` pre-filled with the verified IDs/columns.
5. Print the CanHav hand-off (dashboard URL string to paste).

Verify-only (no prompts, uses whatever IDs are in `.env`):

```bash
npx tsx discover.ts
# or: npm run discover
```

Type-check:

```bash
npm run typecheck
```

## Finding query IDs

Open the chart on the public dashboard — the URL shows `dune.com/queries/<ID>`.

- **Ethena sUSDe / USDe supply** — `dune.com/hildobby/ethena`
- **USD.AI sUSDai supply** — USD.AI's official dashboard (entropy_advisors); ref
  series at `aavescan.com/rates/usdai-susdai`.

## If you upgrade to Plus+

Two things unlock for full end-to-end automation:
- **GitOps repo sync** — a `queries.yml` listing query IDs that Dune syncs to GitHub.
- **Query/dashboard API** — create the supply queries and assemble the dashboard
  programmatically, eliminating the manual UI step.

## Security

- `.env` and `*.key` are gitignored. Never commit your API key.
- If a key is ever exposed, rotate it immediately at the Dune settings page.
